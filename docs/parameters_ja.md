# Irodori-TTS パラメータガイド

このドキュメントでは、Irodori-TTS で使用される主な推論およびトレーニングパラメータについて説明します。

## バージョンに関する注意事項

現在 `main` ブランチは v3 コードベースをターゲットにしています。これは現在の VoiceDesign リリースを含む、v2 チェックポイントとの下位互換性を保っています。

- v3 ベースのチェックポイントには統合された長さ（Duration）予測器が含まれており、`--seconds` を省略した場合に自動的に出力の長さを推定できます。
- v3 ベースリリースのトレーニングは2つのフェーズに分かれています。まず RF/DiT 本体のトレーニング、次に長さ予測器の `duration_only` トレーニングです。
- v2 チェックポイントには長さ予測器が含まれておらず、固定の30秒ターゲットでトレーニングされていますが、引き続き v3 コードでサポートされています。
- VoiceDesign チェックポイントは現在 v2 リリースを使用し、`use_caption_condition: true` を設定して、意図的に話者/参照条件付けを無効にしています。

## 推論パラメータ

### チェックポイントの選択

| パラメータ | デフォルト | 備考 |
|-----------|---------|-------|
| `--checkpoint` | `--hf-checkpoint` が設定されていない場合は必須 | ローカルの `.pt` または `.safetensors` チェックポイント。変換済みのローカルチェックポイントや、ダウンロードしたモデルファイルを直接参照する場合に使用します。 |
| `--hf-checkpoint` | `--checkpoint` が設定されていない場合は必須 | Hugging Face のリポジトリ ID。ランタイムがリポジトリから `model.safetensors` をダウンロードします。 |
| `--lora-adapter` | `None` | 推論時に動的にロードされるオプションの PEFT LoRA アダプターディレクトリ。アダプターはベースチェックポイントにはマージされません。 |
| `--codec-repo` | `Aratako/Semantic-DACVAE-Japanese-32dim` | 参照音声のエンコードと生成された潜在変数のデコードに使用される DACVAE コーデック。通常はチェックポイントのメタデータと一致させる必要があります。 |

`--checkpoint` または `--hf-checkpoint` のいずれかを使用し、両方は指定しないでください。

### テキスト、キャプション、参照の条件付け

| パラメータ | デフォルト | 備考 |
|-----------|---------|-------|
| `--text` | 必須 | 合成するテキスト。チェックポイントのテキストトークナイザーでトークン化されます。 |
| `--caption` | `None` | VoiceDesign チェックポイント用の声質・スタイル制御テキスト。キャプション条件付けのないチェックポイントでは無視されるか効果がありません。 |
| `--ref-wav` | `None` | ベースモデルの話者/スタイル条件付けに使用される参照波形（お手本音声）。 |
| `--ref-latent` | `None` | 推論時に `--ref-wav` をエンコードする代わりに使用される、事前に計算された参照潜在変数 (`.pt`)。同じ参照音声で繰り返し推論する場合に便利です。 |
| `--no-ref` | `False` | 話者/参照条件付けを無効にします。VoiceDesign チェックポイントや、ベースチェックポイントでテキストのみの推論を行う場合に使用します。 |
| `--max-ref-seconds` | `30.0` | エンコード前の参照音声の長さを制限します。リリースされたモデルは最大30秒の音声でトレーニングされているため、デフォルトの制限を維持することをお勧めします。意図的に制限を無効にしたい場合のみ `<=0` に設定してください。 |
| `--ref-normalize-db` | `-16.0` | DACVAE エンコード前の参照音声に適用される音量（ラウドネス）のターゲット。この正規化はコーデックのトレーニング時に使用されたため、デフォルトを維持することをお勧めします。制御された実験を行う場合のみ `none` を使用してください。 |
| `--ref-ensure-max` | `True` | 音量正規化が無効な場合、ピーク振幅が `1.0` を超える場合にのみ参照音声を縮小（スケールダウン）します。通常は、このフォールバックに頼るのではなく、音量正規化を有効にしたままにするのが望ましいです。 |
| `--max-text-len` | チェックポイントのメタデータ または `256` | テキストトークンの最大長。これより長いテキストは切り捨てられます。チェックポイント/トレーニング時の設定を維持することをお勧めします。 |
| `--max-caption-len` | チェックポイントのメタデータ または `max_text_len` | VoiceDesign チェックポイントのキャプショントークンの最大長。チェックポイント/トレーニング時の設定を維持することをお勧めします。 |

参照音声は条件付けのシグナルであり、生成されるターゲットではありません。短くクリーンな参照音声が適している場合もありますが、最も重要なのは音楽、ノイズ、または複数の話者が混ざるのを避けることです。
話者条件付けチェックポイントの場合、同じ話者の参照音声を何度も再利用する時は `--ref-latent` を使うのが最速です。

### 長さの制御（Duration Control）

| パラメータ | デフォルト | 備考 |
|-----------|---------|-------|
| `--seconds` | `None` | 手動の出力時間（秒）。設定した場合、常にデフォルトの長さの動作を上書きします。 |
| `--duration-scale` | `1.0` | 長さ予測が使用される場合、予測された長さに乗算されます。`1.0` を超える値は音声を長くし、`1.0` 未満の値は音声を短くします。 |

推奨される長さの動作は、チェックポイントによって異なります。

- VoiceDesign リリースを含む v2 チェックポイントは、固定の30秒ターゲットでトレーニングされています。異なる長さを設定すると、推論がトレーニング設定から外れるため推奨されません。
- v3 ベースのチェックポイントは、可変長のターゲットと統合された長さ予測器でトレーニングされています。これらのチェックポイントでは、モデルが自動的に長さを選択できるように `--seconds` を設定しないことをお勧めします。正確な制御が必要な場合は、引き続き手動で `--seconds` を利用できます。

`--seconds` が省略された場合、ランタイムはロードされたチェックポイントに長さ予測器の重みが含まれているかを確認します。含まれている場合は、予測されたフレーム数が使用され、さらに `--duration-scale` によってスケーリングされます。含まれていない場合は、ランタイムは30秒にフォールバックします。

### サンプリングと候補生成

| パラメータ | デフォルト | 備考 |
|-----------|---------|-------|
| `--num-steps` | `40` | Euler 積分ステップ数。値を大きくすると遅くなりますが、ある程度まで安定性が向上する可能性があります。 |
| `--t-schedule-mode` | `linear` | RF Euler サンプリングのタイムステップスケジュール。`sway` を使用すると Sway Sampling を有効にします。 |
| `--sway-coeff` | `-1.0` | Sway Sampling 係数。負の値は、ノイズ側により多くのスケジュール解像度を割り当てます。 |
| `--num-candidates` | `1` | 1回のバッチサンプリングパスで生成される候補数。値を大きくすると VRAM の使用量が増加します。 |
| `--decode-mode` | `sequential` | `sequential` は候補を1つずつデコードし、VRAM の使用量を抑えます。`batch` はすべての候補を一緒にデコードし、高速化できる場合があります。 |
| `--seed` | ランダム | サンプリングのシード値。同じチェックポイントとパラメータで再現性のある結果を得たい場合に設定します。 |
| `--truncation-factor` | `None` | サンプリング前の初期ガウスノイズをスケーリングします。`0.8` や `0.9` などの値は変動を減らすことができますが、表現力が低下する可能性もあります。 |
| `--rescale-k` / `--rescale-sigma` | `None` | 一時的なスコアリスケーリングのパラメータ。両方を一緒に設定するか、両方とも未設定のままにしてください。 |

`--num-steps` は通常、品質と速度を調整する際に最初に試す設定です。簡単な実験では低い値でも許容できる場合がありますが、最終的なサンプルの生成時は、他の変更を加える前にデフォルトから始めることをお勧めします。

より低遅延で実験する場合は、ステップ数を減らして Sway Sampling を試してください。

```bash
uv run python infer.py \
  --hf-checkpoint Aratako/Irodori-TTS-500M-v3 \
  --text "こんにちは、私はAIです。これは音声合成のテストです。" \
  --ref-wav path/to/reference.wav \
  --num-steps 6 \
  --t-schedule-mode sway \
  --sway-coeff -1.0 \
  --output-wav outputs/sample_sway.wav
```

### Classifier-Free Guidance (CFG)

| パラメータ | デフォルト | 備考 |
|-----------|---------|-------|
| `--cfg-scale-text` | `3.0` | テキスト条件付けのガイダンス強度。値を高くすると、テキスト条件がより強く強制されます。 |
| `--cfg-scale-caption` | `3.0` | キャプション/スタイル条件付けのガイダンス強度。VoiceDesign チェックポイントに適用されます。 |
| `--cfg-scale-speaker` | `5.0` | 参照話者条件付けのガイダンス強度。話者条件付けが無効な場合は無視されます。 |
| `--cfg-guidance-mode` | `independent` | CFG の形式。`independent`、`joint`、または `alternating` を指定します。 |
| `--cfg-scale` | `None` | 非推奨。すべての有効な CFG スケールの共有上書き設定。条件ごとのスケールパラメータを使用することを推奨します。 |
| `--cfg-min-t` | `0.5` | CFG がアクティブになるタイムステップの下限。 |
| `--cfg-max-t` | `1.0` | CFG がアクティブになるタイムステップの上限。 |

`independent` モードでは、有効な各条件は、単一の大きなバッチ内で独自の無条件（unconditional）ブランチを取得します。これは、異なるテキスト、キャプション、および話者のスケールを使用するための最も柔軟なモードですが、CFG ステップ中のバッチサイズは有効な条件の数に応じて大きくなるため、より多くの VRAM と計算を使用する可能性があります。NFE（Neural Function Evaluations）の観点では、CFG アクティブステップ中は `1 + 有効なCFG条件の数` になります。リリースされたベースおよび VoiceDesign の設定では、一般的に2つの有効な CFG 条件があるため、通常は 3倍 の計算量になります。
`joint` は、すべての有効な条件を一緒にドロップし、等しい CFG スケールを想定します。これは条件付きブランチに加えて、CFG ステップ中に1つの共同の無条件ブランチを使用するため、計算量は 2倍 になります。
`alternating` も CFG ステップごとに1つの無条件ブランチを使用するため 2倍 ですが、どの条件をドロップするかをステップごとに交互に切り替えます。

CFG スケールを上げるとその条件への忠実度が向上する可能性がありますが、値が高すぎると音声が不自然になる場合があります。発音が弱い場合は、`--cfg-scale-text` を少し上げてみてください。話者の類似性が弱い場合は、`--cfg-scale-speaker` または以下の話者 K/V コントロールを試してください。

### 話者 K/V スケーリング

| パラメータ | デフォルト | 備考 |
|-----------|---------|-------|
| `--speaker-kv-scale` | `None` | 話者コンテキストの K/V プロジェクションに適用される追加のスケーリング。`1.0` を超える値は、話者のアイデンティティを強化できる可能性があります。 |
| `--speaker-kv-min-t` | `0.9` | `t >= value` の間のみ、話者 K/V スケーリングを適用します。 |
| `--speaker-kv-max-layers` | `None` | 話者 K/V スケーリングを最初の N 個の拡散レイヤーに制限します。 |

これらのパラメータは、話者の類似性を制御するための実験的な機能です。参照条件付けが有効になっている話者条件付けチェックポイントでのみ意味を持ちます。生成された音声が参照音声から逸脱する場合は、CFG を大きく変更する前に、適度な `--speaker-kv-scale` の値を試してください。

### デバイス、精度、およびコンパイル

| パラメータ | デフォルト | 備考 |
|-----------|---------|-------|
| `--model-device` | auto | RF-DiT モデルのデバイス（例：`cuda`、`mps`、`cpu`）。 |
| `--codec-device` | auto | DACVAE エンコード/デコードのデバイス。`--model-device` と異なっていても構いません。 |
| `--model-precision` | `fp32` | モデルの計算精度：`fp32` または `bf16`。 |
| `--codec-precision` | `fp32` | コーデックの計算精度：`fp32` または `bf16`。 |
| `--compile-model` | `False` | コアな推論メソッドで `torch.compile` を有効にします。コンパイルのため、初回実行は遅くなります。 |
| `--compile-dynamic` | `False` | `torch.compile` で `dynamic=True` を使用します。 |
| `--context-kv-cache` | `True` | サンプリングの高速化のため、テキスト/話者/キャプションの K/V プロジェクションを事前に計算します。 |

CUDA での推論の場合、`bf16` はサポートされている GPU でメモリ使用量を減らし、速度を向上させることができます。CPU または MPS の場合、安全なデフォルトは `fp32` です。`--compile-model` は、同様の形状を持つ多数のリクエストを実行する場合に最も役立ちます。

### テールのトリミングと処理時間

| パラメータ | デフォルト | 備考 |
|-----------|---------|-------|
| `--trim-tail` | `True` | 平坦化ヒューリスティックを使用して、終端のゼロに近い潜在領域をトリミング（切り落とし）します。 |
| `--tail-window-size` | `20` | テールトリミングのヒューリスティックで使用されるウィンドウサイズ。 |
| `--tail-std-threshold` | `0.05` | テールトリミングの標準偏差しきい値。 |
| `--tail-mean-threshold` | `0.1` | テールトリミングの平均値しきい値。 |
| `--show-timings` | `True` | 主要な推論段階の処理時間の内訳を表示します。 |

テールトリミングは主に v2 チェックポイントのために導入されました。v2 は固定で30秒の出力を生成するため、音声の内容の後に未使用の末尾領域が残る可能性があるためです。v3 ベースのチェックポイントはより適切な出力長さを予測するため、この機能の重要性は低くなります。
有効な音声が過剰にトリミングされてしまう場合は、まず `--trim-tail` を無効にしてください。トリミングのヒューリスティックを細かく制御する必要がある場合のみ、テールのしきい値を調整してください。

## トレーニングパラメータ

トレーニングは `model` と `train` のセクションを持つ YAML ファイルで構成されます。CLI オプションは、明示的に指定された場合 YAML の値を上書きします。

### データとチェックポイントのフロー

| パラメータ | デフォルト | 備考 |
|-----------|---------|-------|
| `--config` | `None` | `model` と `train` の上書き設定を含む YAML ファイル。 |
| `--manifest` | 必須 | `prepare_manifest.py` によって生成される JSONL マニフェスト。各行には `text` と `latent_path` を含める必要があり、`speaker_id` と `caption` はモデルに応じてオプションです。 |
| `--output-dir` | `outputs/irodori_tts` | チェックポイント、トレーナーの状態、設定ファイル、ログを保存するディレクトリ。 |
| `--init-checkpoint` | `None` | `.pt` または `.safetensors` チェックポイントからモデルの重みを初期化し、オプティマイザ/スケジューラの状態を最初から開始します。 |
| `--resume` | `None` | トレーニングチェックポイントから完全なトレーニング状態を復元します。フルモデルの実行には `.pt` を使用し、LoRA の実行にはチェックポイントディレクトリを使用します。 |

リリースされた推論用の重みからのファインチューニングには `--init-checkpoint` を使用してください。`--resume` は中断されたトレーニングを再開する場合のみ使用します。

### モデル設定 (Model Config)

| フィールド | 備考 |
|-------|-------|
| `latent_dim` | モデルが想定する DACVAE の潜在変数の次元。v2/v3 500M 設定では `32` を使用します。 |
| `latent_patch_size` | モデルの各トークンごとにグループ化される潜在フレームの数。 |
| `model_dim`, `num_layers`, `num_heads`, `mlp_ratio` | メインの拡散（Diffusion） Transformer の幅、深さ、アテンションヘッド数、および MLP 拡張率。 |
| `text_tokenizer_repo`, `text_vocab_size`, `text_add_bos` | テキストエンコーダーのトークナイザーと語彙の設定。 |
| `text_dim`, `text_layers`, `text_heads`, `text_mlp_ratio` | テキストエンコーダーのサイズ。 |
| `speaker_dim`, `speaker_layers`, `speaker_heads`, `speaker_patch_size`, `speaker_mlp_ratio` | 参照/話者エンコーダーのサイズ。キャプション条件付けによって話者条件付けが無効化されている場合は無視されます。 |
| `use_caption_condition` | VoiceDesign のキャプションパスを有効にし、話者/参照条件付けを無効にします。 |
| `caption_*` フィールド | キャプションエンコーダーのトークナイザーとアーキテクチャ。未設定のままにすると、多くのフィールドが対応するテキスト設定を継承します。 |
| `timestep_embed_dim`, `adaln_rank`, `norm_eps` | 拡散条件付けと正規化のパラメータ。 |
| `use_duration_predictor` | v3 の長さ（Duration）予測を有効にします。 |
| `duration_*` フィールド | 長さ予測器のアーキテクチャ、隠れ層のサイズ、深さ、ドロップアウト、話者条件付け、トークン合計の初期化。 |

アーキテクチャのフィールドは、初期化するチェックポイントと一致している必要があります。次元、レイヤー数、語彙サイズ、または条件付けブランチを変更すると、最初からトレーニングすることを意図している場合や、コードで処理されるアップグレードパスを使用している場合を除き、通常はチェックポイントの読み込みができなくなります。

### バッチサイズ、長さ、マスキング

| パラメータ / フィールド | データクラスのデフォルト | 備考 |
|-------------------|----------------------|-------|
| `batch_size` / `--batch-size` | `8` | プロセスごとのマイクロバッチサイズ。DDP では、グローバルなバッチサイズはワールドサイズ倍になります。 |
| `gradient_accumulation_steps` / `--gradient-accumulation-steps` | `1` | オプティマイザを更新する前に、複数のマイクロバッチにわたって勾配を蓄積します。 |
| `max_text_len` / `--max-text-len` | `256` | テキスト条件付けの最大トークン長。 |
| `max_caption_len` / `--max-caption-len` | `None` | キャプション条件付けの最大トークン長。デフォルトは `max_text_len` です。 |
| `max_latent_steps` / `--max-latent-steps` | `750` | 各サンプルからロードされる最大潜在長のステップ数。25fps の場合、`750` は約30秒になります。 |
| `fixed_target_latent_steps` / `--fixed-target-latent-steps` | `750` | 設定した場合、すべてのトレーニングターゲットはこの長さにパディング/切り捨てられます。可変長トレーニングの場合は YAML で `null` に設定します。 |
| `fixed_target_full_mask` / `--fixed-target-full-mask` | `True` | 固定長トレーニングの場合、パディングされた末尾の位置をロス計算用マスクに含めます。 |
| `rf_loss_mode` / `--rf-loss-mode` | `echo` | RF 損失の正規化。`utterance_mean` は発話ごとの平均を取り、v3 の可変長設定で使用されます。 |

v2 および現在の VoiceDesign の設定は、固定の30秒ターゲットを使用します。v3 のフェーズ1本体とフェーズ2の長さの設定では、`fixed_target_latent_steps: null`、`fixed_target_full_mask: false`、および `rf_loss_mode: utterance_mean` が設定されており、サンプルが本来の長さを保てるようになっています。

### オプティマイザとスケジュール

| パラメータ / フィールド | データクラスのデフォルト | 備考 |
|-------------------|----------------------|-------|
| `precision` / `--precision` | `bf16` | 順伝播（Forward-pass）の計算精度。重みとオプティマイザの状態は FP32 のままです。 |

| `compile_model` / `--compile-model` | `False` | トレーニング中に `torch.compile` を有効にします。 |
| `optimizer` / `--optimizer` | `muon` | `muon` または `adamw`。 |
| `learning_rate` / `--lr` | `1e-4` | 基本学習率。 |
| `weight_decay` / `--weight-decay` | `0.01` | Weight decay を使用するオプティマイザグループの設定値。 |
| `adam_beta1`, `adam_beta2`, `adam_eps` | `0.9`, `0.999`, `1e-8` | AdamW のハイパーパラメータ。 |
| `muon_momentum` / `--muon-momentum` | `0.95` | Muon によって使用されるモメンタム。 |
| `lr_scheduler` / `--lr-scheduler` | `none` | `none`、`cosine`、または `wsd`。 |
| `warmup_steps` / `--warmup-steps` | `0` | 学習率ウォームアップのオプティマイザステップ数。 |
| `stable_steps` / `--stable-steps` | `0` | WSD スケジュールの安定したプラトー長。 |
| `min_lr_scale` / `--min-lr-scale` | `0.1` | 減衰終了時の最小学習率の乗数。 |
| `grad_clip_norm` | `1.0` | 勾配クリッピングノルム。現在は YAML で設定されています。 |

500M のサンプル設定では `optimizer: muon` と `lr_scheduler: wsd` が使用されています。有効なバッチサイズを変更する場合は、学習率とウォームアップの長さも一緒に見直してください。

### 条件ドロップアウトとタイムステップ

| パラメータ / フィールド | データクラスのデフォルト | 備考 |
|-------------------|----------------------|-------|
| `text_condition_dropout` / `--text-condition-dropout` | `0.1` | トレーニング中にテキスト条件付けを null（無条件）に置き換える確率。 |
| `caption_condition_dropout` / `--caption-condition-dropout` | `0.1` | キャプション条件付けに関する同様の確率。 |
| `speaker_condition_dropout` / `--speaker-condition-dropout` | `0.1` | 話者/参照条件付けに関する同様の確率。 |
| `timestep_stratified` / `--timestep-stratified` | `True` | 層化（Stratified）ロジットノーマルなタイムステップサンプリングを使用します。 |
| `timestep_logit_mean`, `timestep_logit_std` | `0.0`, `1.0` | ロジットノーマルなタイムステップ分布のパラメータ。 |
| `timestep_min`, `timestep_max` | `0.001`, `0.999` | タイムステップのサンプリングの下限と上限。 |

条件ドロップアウトは、推論時に Classifier-Free Guidance (CFG) が機能するために必要です。ドロップアウトが低すぎると CFG の動作が弱まる可能性があり、高すぎると条件付けの品質が低下する可能性があります。

### VoiceDesign とキャプションのウォームアップ

| パラメータ / フィールド | データクラスのデフォルト | 備考 |
|-------------------|----------------------|-------|
| `use_caption_condition` | `False` | キャプション条件付けを有効にし、話者/参照条件付けを無効にするモデル設定フィールド。 |
| `caption_warmup` / `--caption-warmup` | `False` | トレーニングの初期段階で、キャプション専用のパラメータのみを更新します。 |
| `caption_warmup_steps` / `--caption-warmup-steps` | `0` | キャプション専用のウォームアップを行うオプティマイザステップ数。 |

`caption_warmup` は、ベースアーキテクチャを VoiceDesign に適合させる際に便利です。キャプションブランチは、通常の共同トレーニングの前に追いつく必要がある場合があるためです。`warmup_steps` は引き続き学習率スケジューラを制御し、`caption_warmup_steps` はキャプションウォームアップフェーズ中にどのパラメータが勾配を受け取るかを制御します。

公開されている VoiceDesign チェックポイントは現在 v2 ベースです。v3 の VoiceDesign チェックポイントがリリースされるまでは、この機能には v2 VoiceDesign の設定を使用してください。

### 長さ予測器 (Duration Predictor)

| パラメータ / フィールド | データクラスのデフォルト | 備考 |
|-------------------|----------------------|-------|
| `use_duration_predictor` | `False` | モデル内で長さ予測を有効にします。 |
| `train_mode` / `--train-mode` | `rf` | `rf` は RF モデルをトレーニングします。`duration_only` は長さ（Duration）に関係ないパラメータをフリーズし、長さ予測器のみをトレーニングします。 |
| `duration_loss_weight` / `--duration-loss-weight` | `0.1` | RF 損失と共同でトレーニングする際の、長さ（Duration）損失の重み。 |
| `duration_speaker_dropout` / `--duration-speaker-dropout` | `0.1` | 長さ予測における話者特徴量のドロップアウト。 |
| `duration_huber_delta` / `--duration-huber-delta` | `0.1` | 対数長さ（log-duration）回帰損失の Huber delta。 |
| `duration_architecture` | `token_sum_adarn_zero_no_aux` | 長さ予測器のアーキテクチャ。 |
| `duration_hidden_dim`, `duration_layers`, `duration_dropout` | `1024`, `3`, `0.1` | 長さ予測器の Residual SwiGLU の幅、深さ、およびドロップアウト。 |
| `duration_attention_heads` | `8` | プールされた長さ（Duration）バリアントで使用されるアテンションヘッド。共有 DP 構築の設定に残されていますが、token-sum フェーズ2設定ではプーリングアテンションを使用しません。 |
| `duration_speaker_fusion` | `adarn_zero` | 話者条件付けモード。`token_sum_adarn_zero_no_aux` は `adarn_zero` を必要とします。 |
| `duration_token_init_frames` | `9.0` | token-sum 長さヘッドのトークンあたりの初期フレーム数。初期予測は概ね `有効なトークン数 * duration_token_init_frames` になります。 |
| `duration_aux_dim` | `14` | データセットパイプラインによって生成される補助的な長さ特徴量のサイズ。token-sum no-aux モデルは、パイプラインの互換性のためにこのテンソルを検証/パスしますが、予測には使用しません。 |

長さのターゲットは `log1p(フレーム数)` であり、ランタイムは推論時に予測を潜在フレームに戻します。v3 ベースリリースでは、この予測器が推論の統合部分として使用されます。メインの RF モデルを更新せずに、長さ予測を追加または改善したい場合は `duration_only` を使用してください。

現在の v3 フェーズ2の長さ設定では、プールされたベクトルの話者フュージョンバリアントに対するアブレーション（切り分け実験）の後、トークンの寄与合計による予測器を使用しています。エンコードされたテキストシーケンスを維持し、Residual SwiGLU ブロックを話者 AdaRN-Zero で条件付けし、`softplus` によってトークンごとの非負のフレーム寄与を予測し、テキストマスクの下でそれらの寄与を合計して `log1p(合計フレーム数)` を返します。

### LoRA ファインチューニング

| パラメータ / フィールド | データクラスのデフォルト | 備考 |
|-------------------|----------------------|-------|
| `lora_enabled` / `--lora` | `False` | PEFT LoRA によるファインチューニングを有効にします。 |
| `lora_r` / `--lora-r` | `16` | LoRA ランク。値を大きくすると、トレーニング可能なキャパシティとチェックポイントのサイズが増加します。 |
| `lora_alpha` / `--lora-alpha` | `32` | LoRA スケーリング係数。 |
| `lora_dropout` / `--lora-dropout` | `0.0` | LoRA レイヤー内のドロップアウト。 |
| `lora_bias` / `--lora-bias` | `none` | PEFT に渡されるバイアス処理。`none`、`all`、または `lora_only` を指定します。 |
| `lora_target_modules` / `--lora-target-modules` | `diffusion_attn` | プリセット名、正規表現、またはカンマ区切りのモジュールサフィックスリスト。 |
| `lora_modules_to_save` / `--lora-modules-to-save` | `auto` | アダプターと一緒に保存される追加のモジュール。`auto` は、v3 の長さ予測機能が有効なモデルに対して `duration_predictor` を保存します。無効にするには `none` を使用します。 |

推論時には、保存されたアダプターのディレクトリを `infer.py --lora-adapter` に渡します。
動的な LoRA ロードでは、`--compile-model` は引き続き無効のままである必要があります。

一般的なプリセット:

- `diffusion_attn`: 小規模で集中的な拡散アテンション（Diffusion attention）の適応。
- `diffusion_attn_mlp`: より広範な拡散への適応。
- `conditioning`: 条件付けプロジェクションの適応。
- `all_attn_mlp`: エンコーダーと拡散ブロック全体にわたる広範な適応。
- `all_linear`: 最大のプリセット。意図的に広範なカバレッジが必要な場合にのみ役立ちます。

LoRA チェックポイントはアダプターディレクトリとして保存されます。変換中にアダプターの重みがベースモデルにマージされるため、エクスポートされた `.safetensors` は直接推論に使用できます。

## チューニングレシピ (Tuning Recipes)

### テキストの再現性を高める (Better Text Adherence)

デフォルトの `--num-steps 40` から始めます。発音やテキストへの追従が弱い場合は、`--cfg-scale-text` を少し高くしてみてください。アーティファクト（ノイズや不自然な音）が増加した場合は、他のガイダンス値を上げる前に、スケールを下げて戻してください。

### 話者の類似性を強める (Stronger Speaker Similarity)

クリーンな参照音声（お手本音声）を使用し、`--ref-normalize-db` を有効にしたままにします。次に、`--cfg-scale-speaker` を適度に増やしてみてください。それでも十分でない場合は、デフォルトの `--speaker-kv-min-t 0.9` と一緒に、適度な `--speaker-kv-scale` をテストしてください。

### 推論の高速化 (Faster Inference)

`--num-steps` を減らし、`--context-kv-cache` を有効のままにし、VRAM に余裕がある場合は `--decode-mode batch` を使用します。サポートされている CUDA GPU では、`--model-precision bf16` および `--codec-precision bf16` を試してください。長さが似ている多数のリクエストを処理する場合は、`--compile-model` を使用します。

### 低 VRAM での推論 (Lower VRAM Inference)

`--decode-mode sequential` を使用し、`--num-candidates 1` を維持し、お使いのデバイスで安定している `fp32`/`bf16` 設定を選択します。`--seconds` の値を大きくしたり、候補の数（生成数）を増やしすぎないようにしてください。

### リリースされた重みのファインチューニング (Fine-Tuning Released Weights)

リリースされた `.safetensors` を使用して `--init-checkpoint` を指定し、最も近い YAML 設定を選択し、意図的にフルモデル更新が必要な場合を除き、まずは LoRA から始めてください。小さなデータセットの場合は、小さな `valid_ratio` で検証を有効にしたままにし、過学習（オーバーフィッティング）を監視してください。
